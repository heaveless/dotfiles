## Overview

Personal dotfiles repository. Each top-level directory maps to a specific location in `~/`.

## How it works

- `make install` creates symlinks: `~/.config/nvim` ‚Üí `dotfiles/nvim`, `~/.agents` ‚Üí `dotfiles/agents`
- `make uninstall` removes the symlinks
- `install.sh` clones the repo (if missing) and calls `make install`

### Plugin categories

| File | Domain | Plugins |
|------|--------|---------|
| `ui.lua` | Interface | rose-pine, lualine, bufferline, oil.nvim |
| `editor.lua` | Editing | snacks (picker+words+notifier+dashboard+statuscolumn+which-key), gitsigns, toggleterm, Comment, mini.pairs |
| `lang.lua` | Languages | treesitter, mason*, nvim-cmp + LuaSnip, lspconfig, conform |
| `ai.lua` | AI/LLM | opencode.nvim |

### Adding a plugin

1. Identify which domain file it belongs to (or create a new `lua/plugins/*.lua`)
2. Add a lazy.nvim spec to that file
3. Each file returns a table of specs: `return { { "author/plugin", opts = {} }, ... }`

- **opencode.nvim**: Bridge to OpenCode CLI. Context injection, prompts, edit approval via diff.

## Agents

Package: `agents/` ‚Üí `~/.agents/`

Contains 22 AI agent skills (Clerk auth, frontend-design, shadcn, vercel-react-best-practices, etc.)
used by OpenCode and nvim's dotagent plugin.

### Skill structure

Each skill is a directory under `skills/` containing:
- `SKILL.md` ‚Äî Main skill definition (YAML frontmatter + markdown instructions)
- Optional: `evals/`, `references/`, `templates/`, `rules/`, `assets/`, `scripts/`

### Adding/removing skills

Skills are managed externally (via OpenCode CLI). The `.skill-lock.json` tracks installed skills.
To update skills in the repo after external changes, re-copy the `~/.agents/` directory.

## Brewfile

Dependencies installed via `brew bundle`:
- `neovim` ‚Äî Editor
- `ripgrep` ‚Äî Telescope dependency
- `fd` ‚Äî Telescope dependency
- `sqlite3` ‚Äî Optional memory database

## Conventions

- **Indentation**: tabs, width 2 (`.editorconfig`)
- **Git**: no `.DS_Store`, no `*.local` files tracked
- **Secrets**: not tracked. API keys via env vars (`DEEPSEEK_API_KEY`)

## Maintenance

```bash
# Update brew deps
brew bundle --file Brewfile

# Refresh lazy.nvim plugins (inside nvim)
:Lazy sync
```
