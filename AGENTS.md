# AGENTS.md

## Overview

Personal dotfiles repository managed with [GNU Stow](https://www.gnu.org/software/stow/).
Each top-level directory is a stow package that gets symlinked to `~/`.

## Structure

```
dotfiles/
├── nvim/           # stow → ~/.config/nvim/
├── agents/         # stow → ~/.agents/
├── Brewfile        # Homebrew dependencies
├── Makefile        # stow/unstow/restow/install
├── install.sh      # Bootstrap script for new machines
└── README.md
```

## How it works

- `make stow` runs `stow -v nvim agents` — creates symlinks from `~/` to the repo
- `make unstow` removes all symlinks
- `make install` bootstraps Homebrew, installs Brewfile deps, then runs stow
- `install.sh` clones the repo (if missing) and calls `make install`

## Nvim

Package: `nvim/.config/nvim/` → `~/.config/nvim/`

Plugin manager: [lazy.nvim](https://github.com/folke/lazy.nvim) (auto-bootstrapped).

### Architecture

```
nvim/.config/nvim/
├── init.lua                  # Entry: sets leader, clipboard, requires config
├── mcpservers.json           # MCP server definitions (empty by default)
├── lazy-lock.json            # Auto-generated by lazy.nvim
└── lua/
    ├── config/
    │   ├── init.lua          # Bootstrap lazy.nvim + apply theme + lazy.setup
    │   ├── options.lua       # vim.opt settings
    │   ├── keymaps.lua       # All keybindings
    │   └── autocmds.lua      # Yank highlight, startup banner
    ├── theme.lua             # Custom minimal black/green theme
    └── plugins/
        ├── ui.lua            # nvim-tree, icons
        ├── editor.lua        # telescope, hop, gitsigns, toggleterm
        ├── lang.lua          # treesitter, mason, lspconfig, conform
        └── ai.lua            # codecompanion (DeepSeek), dotagent, mcphub
```

### Plugin categories

| File | Domain | Plugins |
|------|--------|---------|
| `ui.lua` | Interface | nvim-tree, nvim-web-devicons |
| `editor.lua` | Editing | telescope (fzf-native), hop, gitsigns, toggleterm |
| `lang.lua` | Languages | treesitter, mason, mason-lspconfig, nvim-lspconfig, conform |
| `ai.lua` | AI/LLM | codecompanion (DeepSeek), dotagent, mcphub |

### Adding a plugin

1. Identify which domain file it belongs to (or create a new `lua/plugins/*.lua`)
2. Add a lazy.nvim spec to that file
3. Each file returns a table of specs: `return { { "author/plugin", opts = {} }, ... }`

### Theme

Custom minimal theme in `lua/theme.lua`. Black background (`#000000`), green foreground (`#00ED8A`).
Applied in `lua/config/init.lua` before lazy.nvim loads plugins.

### AI integration

- **codecompanion.nvim**: Chat with DeepSeek (`deepseek-v4-pro`) via OpenAI-compatible adapter
  - Requires `DEEPSEEK_API_KEY` env var
- **dotagent.nvim**: Skills-based agent using skills from `~/.agents/skills/`
  - Also requires `DEEPSEEK_API_KEY`
- **mcphub.nvim**: MCP server manager, reads config from `~/.config/nvim/mcpservers.json`

Important keymaps:
- `<leader>a` — CodeCompanion chat
- `<leader>s` — DotAgent
- `<leader>mc` — MCP Hub

## Agents

Package: `agents/.agents/` → `~/.agents/`

Contains 22 AI agent skills (Clerk auth, frontend-design, shadcn, vercel-react-best-practices, etc.)
used by OpenCode and nvim's dotagent plugin.

### Skill structure

Each skill is a directory under `skills/` containing:
- `SKILL.md` — Main skill definition (YAML frontmatter + markdown instructions)
- Optional: `evals/`, `references/`, `templates/`, `rules/`, `assets/`, `scripts/`

### Adding/removing skills

Skills are managed externally (via OpenCode CLI). The `.skill-lock.json` tracks installed skills.
To update skills in the repo after external changes, re-copy the `~/.agents/` directory.

## Brewfile

Dependencies installed via `brew bundle`:
- `stow` — Symlink manager
- `neovim` — Editor
- `ripgrep` — Telescope dependency
- `fd` — Telescope dependency
- `sqlite3` — Optional memory database

## Conventions

- **Indentation**: tabs, width 2 (`.editorconfig`)
- **Git**: no `.DS_Store`, no `*.local` files tracked
- **Secrets**: not tracked. API keys via env vars (`DEEPSEEK_API_KEY`)
- **Stow packages**: each top-level dir mirrors the target home directory structure

## Maintenance

```bash
# Update brew deps
brew bundle --file Brewfile

# Refresh lazy.nvim plugins (inside nvim)
:Lazy sync

# Restow after changes
make restow
```
